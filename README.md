# CI Knowledge — Consumer Industries Knowledge Base for Claude Code

A Claude Code plugin that surfaces the Consumer Industries team's collective knowledge from Google Drive. Contributors drop files into a shared folder. Anyone with Claude Code can instantly query that knowledge, browse it, and find out who knows the most about a topic.

Built on the "Team of Teams" framework: Trust, Common Purpose, Shared Consciousness, Empowered Execution.

## How It Works

```
Contributors                    System                         Consumers
───────────                    ──────                         ─────────
Drop files in        →    /ci-refresh scans Drive        →    /ci-ask reads TOC
Google Drive              diffs against _TOC.json             fetches relevant docs
                          Claude generates summaries          synthesizes answers
                          writes _TOC.json to Drive          cites sources + experts
```

**For contributors:** Drop documents into the shared Google Drive folder. That's it.

**For consumers:** Type a slash command in Claude Code to query the team's knowledge.

## Prerequisites

Before installing, make sure you have:

1. **Claude Code** — installed and working ([install guide](https://docs.anthropic.com/en/docs/claude-code))
2. **Google Drive access** — you need access to the shared CI Knowledge Base folder in Google Drive
3. **`fe-google-tools` plugin** — provides Google Drive authentication

If you don't have `fe-google-tools`, install it with vibe:
```bash
vibe plugins
```
Select `fe-google-tools` from the interactive list and enable it.

## Installation

### Step 1: Install the plugin

```bash
# Add the repo as a marketplace source
claude plugin marketplace add datasciencemonkey/ci-knowledge

# Install the plugin
claude plugin install ci-knowledge
```

### Step 2: Authenticate with Google Drive

Open Claude Code and run:
```
/google-auth
```

This opens a browser window. Sign in with your Databricks Google account and click "Allow". You only need to do this once — the token refreshes automatically.

### Step 3: Configure the Drive folder ID

Copy the example config and set your folder ID:

```bash
cp config.example.json config.json
```

Edit `config.json` and set `drive_folder_id` to the shared Google Drive folder ID:

```json
{
  "drive_folder_id": "YOUR_GOOGLE_DRIVE_FOLDER_ID_HERE",
  "google_auth_path": "~/.claude/plugins/cache/fe-vibe/fe-google-tools/1.4.0/skills/google-auth/resources"
}
```

`config.json` is gitignored — your folder ID never gets pushed to the repo.

#### How to find the Google Drive folder ID

1. Open the shared CI Knowledge Base folder in Google Drive
2. Look at the URL in your browser:
   ```
   https://drive.google.com/drive/folders/1A2B3C4D5E6F7G8H9I0J
                                          ^^^^^^^^^^^^^^^^^^^^^^^^
                                          This is the folder ID
   ```
3. Copy the string after `/folders/` — that's the ID
4. Paste it into `config.json` as the `drive_folder_id`

### Step 4: Run the initial sync

```
/ci-refresh
```

This scans the Drive folder, reads each document, generates summaries and tags, and creates `_TOC.json`. The first sync takes a bit longer — subsequent syncs only process new or changed files.

## Skills

| Skill | What it does |
|---|---|
| `/ci-refresh` | Sync the knowledge base with Google Drive. Detects new, modified, and deleted files. |
| `/ci-ask "question"` | Ask a question. Returns an answer with citations and who to talk to. |
| `/ci-browse` | Browse the knowledge base by folder, contributor, or tag. |
| `/ci-expert "topic"` | Find who knows the most about a topic. |

## Usage Examples

### Ask a question
```
/ci-ask "how do we talk to retailers about lakehouse architecture?"
```
Returns a synthesized answer citing specific documents, plus who to talk to for deeper context.

### Find an expert
```
/ci-expert "real-time inventory analytics"
```
Returns a ranked list of people who have written the most about that topic.

### Browse the knowledge base
```
/ci-browse                          # folder overview with doc counts
/ci-browse "Talk Tracks"            # list files in a folder
/ci-browse --author "lorraine"      # all docs by a contributor
/ci-browse --tag "retail"           # all docs with a specific tag
```

### Refresh the knowledge base
```
/ci-refresh
```
Run this when new documents have been added to Drive. It only processes new or changed files — unchanged files are skipped.

## Recommended Folder Structure

Set up the shared Google Drive folder with this structure. Contributors can add new folders at any time — the system discovers them automatically.

```
CI Knowledge Base/
├── _TOC.json              (auto-generated by /ci-refresh)
├── Account Strategies/    customer-specific plans and context
├── Playbooks/             how we run specific motions
├── Outcome Maps/          use case → outcome frameworks
├── Talk Tracks/           how to talk to customers about X
├── Meeting Notes/         call summaries, QBR notes
└── Best Practices/        tribal knowledge, how-we-do-things
```

## How It Works (Technical)

The system is built on a simple table-of-contents pattern:

- `_TOC.json` is a lightweight index stored in Google Drive alongside the knowledge files
- It contains file metadata, AI-generated summaries, and topic tags for each document
- When you ask a question, Claude reads the TOC, identifies the most relevant documents, fetches them from Drive, and synthesizes an answer
- Expert attribution comes from Google Drive file ownership — whoever created the document is listed as the author
- If the TOC hasn't been refreshed in 24+ hours, skills will nudge you to run `/ci-refresh`
- The plugin is **read-only** — it cannot delete, modify, or move any file in Google Drive. The only file it writes is `_TOC.json`.

No databases. No vector stores. No external services. Just Claude Code + Google Drive.

## Troubleshooting

### "You are not authenticated with Google"
Run `/google-auth` and complete the browser sign-in flow.

### "No _TOC.json found"
Run `/ci-refresh` to create the initial index.

### "Knowledge base was last synced X days ago"
Run `/ci-refresh` to pick up recent changes.

### "No matching knowledge found"
The knowledge base may not have documents on that topic yet. Try `/ci-browse` to see what's available, or add a document to the Drive folder.

### Config not found
Make sure `config.json` exists in the plugin root with your `drive_folder_id` set.

## Contributing Knowledge

Anyone with edit access to the shared Google Drive folder can contribute:

1. Create or upload a document to the appropriate subfolder
2. Use descriptive filenames (e.g., "Retail Lakehouse Pitch Strategy.md" not "notes.txt")
3. Ask someone to run `/ci-refresh` (or run it yourself) to pick up the new doc
4. The system auto-generates a summary and tags — no extra work needed
