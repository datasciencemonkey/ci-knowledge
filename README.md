# CI Knowledge — Consumer Industries Knowledge Base for Claude Code

A Claude Code plugin that surfaces the Consumer Industries team's collective knowledge from Google Drive. Contributors drop files into a shared folder. Anyone with Claude Code can instantly query that knowledge, browse it, and find out who knows the most about a topic.

Built on the "Team of Teams" framework: Trust, Common Purpose, Shared Consciousness, Empowered Execution.

## How It Works

```
Contributors                    System                         Consumers
───────────                    ──────                         ─────────
Drop files in        →    /ci-refresh scans Drive        →    /ci-ask reads TOC
Google Drive              diffs against _TOC.json             fetches relevant docs
                          Claude generates summaries          synthesizes answers
                          writes _TOC.json to Drive          cites sources + experts
```

**For contributors:** Drop documents into the shared Google Drive folder. That's it.

**For consumers:** Use Claude Code slash commands to query the knowledge base.

## Skills

| Skill | Description |
|---|---|
| `/ci-refresh` | Sync the knowledge base TOC with Google Drive. Detects new, modified, and deleted files. |
| `/ci-ask "question"` | Query the knowledge base. Returns answers with citations and expert attribution. |
| `/ci-browse` | Browse by folder, contributor, or tag. |
| `/ci-expert "topic"` | Find who knows the most about a topic, ranked by doc authorship. |

## Setup

### 1. Install the plugin

```bash
claude plugin add github:datasciencemonkey/ci-knowledge
```

### 2. Authenticate Google Drive

```bash
/google-auth
```

Requires the `fe-google-tools` plugin for Google Drive authentication.

### 3. Configure the Drive folder

Edit the `FOLDER_ID` in each skill's SKILL.md to point to your team's shared Google Drive folder.

### 4. Create the folder structure

Set up a shared Google Drive folder with this recommended structure:

```
CI Knowledge Base/
├── _TOC.json              (auto-generated by /ci-refresh)
├── Account Strategies/
├── Playbooks/
├── Outcome Maps/
├── Talk Tracks/
├── Meeting Notes/
└── Best Practices/
```

Contributors can add new folders at any time. The system discovers them automatically.

### 5. Initial sync

```
/ci-refresh
```

This scans the Drive folder, generates summaries for all documents, and creates `_TOC.json`.

## Usage Examples

### Ask a question
```
/ci-ask "how do we talk to retailers about lakehouse architecture?"
```

Returns a synthesized answer citing specific documents, plus who to talk to for deeper context.

### Find an expert
```
/ci-expert "real-time inventory analytics"
```

Returns ranked list of people who have authored the most relevant documents.

### Browse the knowledge base
```
/ci-browse                          # folder overview
/ci-browse "Talk Tracks"            # list files in a folder
/ci-browse --author "lorraine"      # docs by a contributor
/ci-browse --tag "retail"           # docs matching a tag
```

## Supported File Types

| Type | How It's Read |
|---|---|
| Google Docs | Exported as plain text via Drive API |
| PDFs | Exported as text via Drive API |
| DOCX | Exported as text via Drive API |
| Markdown (.md) | Read directly |
| HTML (.html) | Read directly |
| Plain text | Read directly |
| Google Sheets | Exported as CSV (first sheet) |
| Images/Videos | Listed in TOC, no summary |

## Architecture

The system is built on a simple table-of-contents pattern:

- `_TOC.json` is a lightweight manifest stored in Google Drive alongside the knowledge files
- It contains file metadata, AI-generated summaries, and topic tags
- Skills read the TOC to find relevant documents, then fetch full content on demand
- Expert attribution comes from Google Drive file ownership metadata
- Staleness detection nudges users to run `/ci-refresh` if the TOC is >24 hours old

No databases. No vector stores. No external services. Just Claude Code + Google Drive.
